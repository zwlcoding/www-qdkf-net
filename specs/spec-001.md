## 1. 项目概述

### 1.1 目标
美化渊渟集团官网 (www-qdkf-net)，采用东方禅意极简设计风格，传达"执经衡门，渊渟岳立"的品牌精神。

### 1.2 核心设计理念
- **禅意**：留白、宁静、意境
- **极简**：去除冗余，聚焦本质
- **东方**：传统美学与现代设计的融合

## 3. 页面结构

### 3.1 整体布局

│                                             │
├─────────────────────────────────────────────┤
│                                             │
│           哲学理念区                         │ 三列卡片，核心理念
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│           管理风格区                         │ 对比展示，左右交替
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│              页脚 (Footer)                   │ 版权、备案信息

### 3.2 导航栏 (Navbar)
**功能**：
- 品牌 Logo + 名称"渊渟"
- 导航链接：首页、理念、管理、联系
- 移动端：汉堡菜单

**样式**：
- 固定定位 `position: fixed`
- 背景透明，滚动后背景变为 `rgba(250, 248, 245, 0.95)` + `backdrop-filter: blur(10px)`
- 高度：PC端 80px，移动端 60px

**交互**：
- 导航链接悬停：底部出现朱砂色下划线，宽度从0到100%动画
- 移动端菜单展开：从右侧滑入

### 3.3 Hero 主视觉区
**功能**：
- 展示品牌名称"渊渟"
- 副标题："执经衡门，渊渟岳立"
- 简短品牌介绍
- 向下滚动指示器

**样式**：
- 全屏高度 `min-height: 100vh`
- 内容垂直居中
- 背景：宣纸白纯色，或极淡的水墨纹理

**交互**：
- 文字淡入动画，依次出现
- 向下箭头上下浮动动画

### 3.4 哲学理念区
**功能**：
- 展示三个核心理念卡片
- 卡片1：静水流深（渊）
- 卡片2：蓄力待发（渟）
- 卡片3：过程体验（经营哲学）

**样式**：
- 三列网格布局，移动端单列
- 卡片：无边框，纯白背景，轻微阴影
- 卡片内：图标 + 标题 + 描述

**交互**：
- 卡片悬停：轻微上浮 + 阴影加深
- 进入视口时依次淡入

### 3.5 管理风格区
**功能**：
- 对比展示传统 vs 渊渟的管理风格
- 4组对比项，左右交替布局

**对比内容**：
| 传统公司 | 渊渟集团 |
|---------|---------|
| KPI驱动 | "别太累，差不多就行" |
| 996福报 | 到点赶人下班 |
| 降本增效 | 增本降效（资源管够，慢慢做） |
| 结果导向 | 过程体验导向 |

**样式**：
- 左右交替布局（文字左/图片右，然后交换）
- 移动端改为上下布局
- 使用朱砂色高亮关键词

**交互**：
- 进入视口时从两侧滑入

### 3.6 页脚 (Footer)
**功能**：
- 版权信息：© 2026 渊渟集团
- 备案信息：粤ICP备18070331号-1
- 可选：社交媒体链接

**样式**：
- 背景：墨黑 `#1A1A1A`
- 文字：宣纸白 `#FAF8F5`
- 内边距：60px 上下

## 4. 响应式设计

### 4.1 断点
- **移动端**：`< 768px`
- **平板**：`768px - 1024px`
- **PC端**：`> 1024px`

### 4.2 适配规则
| 元素 | PC端 | 移动端 |
|-----|------|--------|
| 导航 | 横向链接 | 汉堡菜单 |
| Hero | 大标题居中 | 字号缩小，padding调整 |
| 理念卡片 | 三列 | 单列堆叠 |
| 管理风格 | 左右交替 | 上下堆叠 |
| 容器宽度 | 1200px max | 100% + 20px padding |

## 5. 动效规范

### 5.1 原则
- **纯 CSS 动效**，不使用 JavaScript 动画库
- 优先考虑性能和流畅度
- 尊重用户偏好 `prefers-reduced-motion`

### 5.2 动效列表
| 动效 | 实现方式 | 时长 | 缓动函数 |
|-----|---------|------|---------|
| 导航背景过渡 | CSS transition | 300ms | ease |
| 链接下划线 | CSS transition + transform | 300ms | ease-out |
| Hero文字淡入 | CSS animation + delay | 800ms | ease-out |
| 箭头浮动 | CSS animation | 2s | ease-in-out infinite |
| 卡片悬停上浮 | CSS transition | 300ms | ease |
| 滚动淡入 | CSS animation + IntersectionObserver | 600ms | ease-out |
| 对比区滑入 | CSS animation | 600ms | ease-out |

### 5.3 性能优化
- 使用 `transform` 和 `opacity` 代替 `top/left`
- 添加 `will-change` 属性
- 支持 `prefers-reduced-motion: reduce`
